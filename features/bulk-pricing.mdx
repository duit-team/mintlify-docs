**Bulk Pricing** lets you set a different price, display name, or availability for products in a specific sales context. Common use cases include delivery platform pricing, happy-hour menus, or location-specific price adjustments.

</Frame>

## How It Works

A **context** is simply a combination of a **Location** and/or an **Order Mode** you already have set up — there is no separate "Context" list to create or manage. You pick the combination from the dropdowns at the top of the page (the same selector shown in Menu Builder), for example:
- **Location: Outlet B, Order Mode: All** — a different price point for one branch, across every order mode.
- **Location: All, Order Mode: Delivery** — a higher price to account for delivery platform commission, at every location.
- **Location: Outlet B, Order Mode: Delivery** — both combined, for the most specific override.

Each row in the bulk pricing grid represents one product. You can override:

| Field | What it changes |
|-------|----------------|
| **Price** | The price charged in this context only. Leave blank to use the base product price |
| **Name** | The display name shown to the customer (e.g. `Large Meal Deal`). Leave blank to use the base product name |
| **Active** | Whether the product is available in this context. Toggle off to hide it without affecting the base menu |

Changes here do **not** affect the base product — only the selected context.

<Tip>
  Leave a field blank to inherit the default value from the base product. You only need to fill in the fields that differ from the standard menu for this context.
</Tip>

## Override Priority

You can set up multiple overrides for the same product across different contexts. When an order comes in, the most specific match wins:

1. **Exact match** — a specific Location **and** a specific Order Mode.
2. **Location-only** — a specific Location, Order Mode set to All.
3. **Order Mode-only** — Order Mode set to a specific mode, Location set to All.
4. **Global** — Location and Order Mode both set to All — applies everywhere nothing more specific exists.

If no override matches at all, the base product price, name, and availability are used as-is.

## Apply Overrides

<Steps>
  <Step title="Go to Menu Builder → Bulk Pricing">
    Select the context (e.g. *Delivery*, *Happy Hour*) from the dropdown at the top of the page.
  </Step>
  <Step title="Find the product">
    Use the search bar or scroll to find the product row you want to override.
  </Step>
  <Step title="Enter overrides">
    Click into the price or name cell and type the override value. Toggle the active switch to hide a product in this context.
  </Step>
  <Step title="Save">
    Click **Save** at the top of the page. All changes for this context are saved together.
  </Step>
</Steps>

<Warning>
  Clicking **Save** saves all pending changes on the page at once. If you make changes across multiple rows and navigate away without saving, all unsaved changes are lost. Save frequently when making bulk edits.
</Warning>

## Viewing Overrides in the Menu Builder

Use the **Context** dropdown at the top of the Menu Builder to preview how the menu looks with a specific context applied. This lets you verify your pricing overrides without going live.

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="How do I create a new context?">
    You don't create contexts separately — a context is just a Location and/or Order Mode you already have. Pick the combination you want from the dropdowns at the top of the page. If the combination you need isn't available, make sure the Location (under Manage → Locations) or Order Mode (under Manage → Order Modes) exists first.
  </Accordion>
  <Accordion title="If I change the base product price, do bulk pricing overrides update automatically?">
    No. Override prices are stored independently. If you lower a product's base price but a bulk pricing context has an override, the context will continue to use the override price. You must manually update the override if you want it to reflect the new base price.
  </Accordion>
  <Accordion title="Can I hide an entire category from a specific context?">
    Currently, bulk pricing works at the product level. To hide a full category in a context, toggle the Active switch off for every product in that category within the context grid.
  </Accordion>
  <Accordion title="Does bulk pricing affect the tax calculation?">
    Tax is calculated on the effective price (the override price, if set). If you set a delivery price of RM 15.00 (vs base RM 12.00), tax is calculated on RM 15.00 for delivery orders.
  </Accordion>
</AccordionGroup>
